# AutoDL Pod Setup — Mihomo + Claude Code

## Phase 1: Root — 创建用户

```bash
useradd -m -s /bin/bash cc
usermod -aG sudo cc
chown -R cc:cc /home/cc
```

## Phase 2: Root — 安装 Mihomo

```bash
# 下载 mihomo
cd /tmp
curl -L --connect-timeout 30 --max-time 600 --retry 3 "https://gh-proxy.com/https://github.com/MetaCubeX/mihomo/releases/download/v1.19.22/mihomo-linux-amd64-v1.19.22.gz" -o mihomo.gz
gzip -d mihomo.gz
chmod +x mihomo

mkdir -p ~/mihomo
mv /tmp/mihomo ~/mihomo/mihomo

~/mihomo/mihomo -v
```

## Phase 4: Root — 配置并启动 Mihomo

```bash
unset http_proxy https_proxy

mkdir -p ~/.config/mihomo
cp /home/cc/config.yaml /root/.config/mihomo/config.yaml
wc -c /root/.config/mihomo/config.yaml  # 确认非空

cd ~/.config/mihomo
curl -L --connect-timeout 30 --max-time 120 "https://gh-proxy.com/https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/geoip.metadb" -o geoip.metadb
curl -L --connect-timeout 30 --max-time 120 "https://gh-proxy.com/https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/geosite.dat" -o geosite.dat

# 前台测试一下能不能启动（Ctrl+C 退出）

# 后台运行
nohup ~/mihomo/mihomo -d ~/.config/mihomo > /tmp/mihomo.log 2>&1 &

# 验证（可新开终端）

curl -s -x http://127.0.0.1:7890 https://www.google.com -o /dev/null -w "%{http_code}"
```

## Phase 5: cc 用户 — 安装 Claude Code

mihomo 在 root 下运行，监听 127.0.0.1:7890，所有用户都能用。
`su - cc` 会重置环境变量，所以必须先设 proxy 再装 Claude Code。

```bash
su - cc

# ⚠️ 必须先设代理，否则 claude.ai 会返回 "App unavailable in region"
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890

# 安装 Claude Code
curl -fsSL https://claude.ai/install.sh | bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
claude --version

# 写入所有环境变量到 .bashrc（持久化）
cat >> ~/.bashrc << 'EOF'
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
export ANTHROPIC_BASE_URL="https://api.ikuncode.cc"
export ANTHROPIC_AUTH_TOKEN="sk-fumg5WK5yHWYV4OJaabjrABvQvO2eslfMda47gPtUs5iV9QX"
export ANTHROPIC_MODEL="claude-opus-4-6"
EOF
source ~/.bashrc

# 配置 Claude Code 权限
mkdir -p ~/.claude
cat > ~/.claude/settings.json << 'EOF'
{
  "permissions": {
    "defaultMode": "bypassPermissions"
  }
}
EOF

# 验证
claude --version
```

## Phase 12: Root — 挂载数据盘给 cc 用户

AutoDL 数据盘在 `/root/autodl-tmp`（500G），cc 用户无权访问 `/root/`。
通过 bind mount 映射到 cc 可访问的路径。

```bash
# 以 root 执行
mkdir -p /home/cc/data
mount --bind /root/autodl-tmp /home/cc/data
chown cc:cc /home/cc/data
echo '/root/autodl-tmp /home/cc/data none bind 0 0' >> /etc/fstab

# 验证（切到 cc 用户测试）
su - cc -c "ls /home/cc/data && touch /home/cc/data/test && rm /home/cc/data/test && echo OK"

chmod o+x /root
rmdir /home/cc/data
ln -s /root/autodl-tmp /home/cc/data
chown -R cc:cc /root/autodl-tmp
su - cc -c "df -h /home/cc/data && ls /home/cc/data/ && touch /home/cc/data/.test && rm /home/cc/data/.test && echo 'ALL OK'"
```

> ⚠️ bind mount 重启后失效。如需持久化，以 root 执行：
> echo '/root/autodl-tmp /home/cc/data none bind 0 0' >> /etc/fstab

## Mihomo 故障排查与节点管理

### 停止 Mihomo

```bash
# root 执行
pkill mihomo
ps aux | grep mihomo   # 确认已停止
unset http_proxy https_proxy
```

### 方案 A：proxy-providers 自动订阅（推荐）

不需要手动下载 config.yaml，mihomo 自动通过在线转换器拉取并解析订阅。以 root 执行：

```bash
cat > /root/.config/mihomo/config.yaml << 'MEOF'
mixed-port: 7890
external-controller: 127.0.0.1:9090
mode: rule
log-level: info

proxy-providers:
  my-sub:
    type: http
    url: "https://api.dler.io/sub?target=clash&url=https%3A%2F%2Fdash.pqjc.site%2Fapi%2Fv1%2Fpq%2F8beec87101a19033dceaed20133bd993&config=https%3A%2F%2Fraw.githubusercontent.com%2FACL4SSR%2FACL4SSR%2Fmaster%2FClash%2Fconfig%2FACL4SSR_Online.ini"
    interval: 3600
    path: ./proxy-provider.yaml
    health-check:
      enable: true
      url: http://www.gstatic.com/generate_204
      interval: 300

proxy-groups:
  - name: PROXY
    type: select
    use:
      - my-sub

rules:
  - GEOIP,CN,DIRECT
  - MATCH,PROXY
MEOF
```

验证转换器是否成功返回了节点（启动 mihomo 之前先检查）：

```bash
# 直接请求转换器，看返回的是不是 YAML（开头应是 port: 或 proxies:）
curl -s "https://api.dler.io/sub?target=clash&url=https%3A%2F%2Fdash.pqjc.site%2Fapi%2Fv1%2Fpq%2F8beec87101a19033dceaed20133bd993&config=https%3A%2F%2Fraw.githubusercontent.com%2FACL4SSR%2FACL4SSR%2Fmaster%2FClash%2Fconfig%2FACL4SSR_Online.ini" | head -20
```

> 正确输出示例：开头是 `port: 7890` 或 `proxies:` 开头的 YAML，且能看到节点列表。
> 错误输出：空白、HTML 错误页、base64 乱码、或超时无响应 → 换备用转换器。

> ⚠️ 如果 `api.dler.io` 从 AutoDL 访问不了，把 URL 中的 `api.dler.io/sub` 替换为：
> - `sub.xeton.dev/sub`
> - `sub.id9.cc/sub`

### 方案 B：本地转换后上传（备用）

AutoDL 机房 IP 可能被订阅商或转换器 403，此时在本地 Windows PowerShell 下载：

```powershell
# 方法1：订阅商自带 clash 参数
curl.exe -L -k "https://dash.pqjc.site/api/v1/pq/8beec87101a19033dceaed20133bd993?clash=1" -o "d:\DARE\.context\config.yaml"

# 方法2：用 subconverter 公共 API 转换
$sub = [uri]::EscapeDataString("https://dash.pqjc.site/api/v1/pq/8beec87101a19033dceaed20133bd993")
curl.exe -L "https://api.dler.io/sub?target=clash&url=$sub&config=https://raw.githubusercontent.com/ACL4SSR/ACL4SSR/master/Clash/config/ACL4SSR_Online.ini" -o "d:\DARE\.context\config.yaml"
```

> 下载后用记事本打开确认：开头应是 `port:` 或 `mixed-port:` 的 YAML 格式，不是 base64 乱码。
> 确认后通过 JupyterLab 上传到 `/home/cc/config.yaml`，然后 root 执行：

```bash
cp /home/cc/config.yaml /root/.config/mihomo/config.yaml
```

### 启动 Mihomo 并验证

```bash
# root 执行
pkill mihomo 2>/dev/null
nohup ~/mihomo/mihomo -d ~/.config/mihomo > /tmp/mihomo.log 2>&1 &

# 验证 API 可达
curl -s http://127.0.0.1:9090/

# 查看可用节点数和当前选中节点（方案A 用 PROXY，方案B 用 GLOBAL）
curl -s http://127.0.0.1:9090/proxies/PROXY | python3 -c "import sys,json; d=json.load(sys.stdin); print(f'节点数: {len(d[\"all\"])}\n当前: {d[\"now\"]}')"

# 测试所有节点延迟（只有响应的才可用，超时的不会出现）
curl -X GET "http://127.0.0.1:9090/group/PROXY/delay?url=http://www.gstatic.com/generate_204&timeout=5000"
```

> 2026-04-08 实测延迟参考（AutoDL 机房）：
> - 日本东京 146ms（最快）、日本东京4 168ms
> - 美国02 175ms、美国05 177ms、美国08 183ms
> - 新加坡4 252ms、新加坡3 263ms
> - 香港节点普遍 700-1500ms（不推荐）
>
> 推荐：Claude API 用美国节点（避免区域限制），一般上网用日本节点（最低延迟）
