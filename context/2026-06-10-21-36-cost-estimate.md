# Cost Estimation

## Dataset

ABC (A Big CAD Model Dataset for Geometric Deep Learning, Koch et al., CVPR 2019)

<https://deep-geometry.github.io/abc-dataset/>

1M STEP files, ~500GB, unzip 建议留 700GB 空间

## Multi-View Rendering

```bash
STEP file → Blender headless 加载 → 居中 + 归一化缩放
                                              ↓
                                设置 30 个相机位 (10方位角 × 3仰角)
                                              ↓
                                    EEVEE GPU 渲染 (逐视角)
                                              ↓
                                  224×224 JPEG q90 彩色图 × 30
```

30-view = 10 方位角 × 3 仰角, image_size 224 × 224

- 方位角: 0°, 36°, 72°, 108°, 144°, 180°, 216°, 252°, 288°, 324°
- 仰角: 15°, 35°, 60°

1 x RTX 5090 耗时预估 ~60h

1M x 30-view = 30M JPEG images, ~625GB

## Text Annotation

```bash
渲染图 (30-view 全部) → Qwen3.6-27B (multi-image input) → shape-centric caption
                                                                ↓
                                                           1M x 1 JSON
```

┌───────────────────────┬─────────────────────────────┐
│ Component             │ Source                      │
├───────────────────────┼─────────────────────────────┤
│ VLM                   │ HF QwenLM/Qwen3.6-27B       │
└───────────────────────┴─────────────────────────────┘

Prompt 模板 (shape-centric, 参考 partABC (NURBGen arXiv 2511.06194) 300K captions 风格):
"Describe this CAD part's geometry in one sentence.
Focus on shape, structure, and key features (holes, threads, flanges).
Example: a flat cylindrical disc with a central through-hole and six evenly spaced mounting holes."

27B fp16 ~54GB, 需双卡 (2 x RTX 5090)

~7680 visual tokens/call 耗时预估 ~16h

1M x 1 JSON ~1GB

## Quality Filtering (preprocessing 前置)

目标: 把"质量筛选"从渲染 timeout 的隐式副产物, 提到渲染之前, 变成显式、廉价、可解释、可复现的一步。
教训: 上次执行渲染卡在 276K/990K, 剩余 STEP "太复杂渲染不动", 被迫把训练阈值从 900K 硬降到 340K
—— 即 "能否在 30s 内 mesh 完" 偷偷变成了筛选规则。本节要消除这种隐式筛选。

核心原则: 在 mesh (BRepMesh, 最贵且超时所在) 之前, 用便宜的拓扑探查预测哪些会爆炸, 提前分流。
筛选不删除文件, 只往 manifest 写 `quality_flags` 列 —— 被筛掉的可随时查询、复核、是否误杀。

筛选方法分为 4 个方面 (按成本递增, 逐一确认):

### 方面 1: 元数据级筛选 (零解析成本, 不打开文件)

- file_size: 过大 (如 > 50MB) → `oversized` (几何爆炸, mesh 必超时)
- file_size: 过小 / 空 / 截断 → `corrupt`
- ABC 自带 metadata (*_features.yml / stats): 若下载时带了, 直接读面数/边数, 连 STEP 都不解析

### 方面 2: 拓扑探查筛选 (秒级, 不 mesh —— 筛选主力)

STEPControl_Reader.ReadFile + 拓扑遍历 (TopExp_Explorer), 不调 BRepMesh:

- ReadFile != 1 → `step_read_fail`
- n_faces 极大 (如 > 5000) → `too_complex` (mesh 必超时)
- n_faces 极小 (如 < 3) → `too_simple` (退化/平板/空壳, 无信息量)
- n_solids == 0 → `no_solid` (纯曲面/线框, 渲染无意义)
- bbox 某维 ≈ 0 → `degenerate` (退化成面/线)
- bbox 长宽比极端 (如 > 1000:1) → `degenerate` (细长件, 渲染全黑/全白)

### 方面 3: 渲染后图像级筛选 (廉价收尾, 可选)

少数 mesh 成功但视觉退化的漏网:

- 前景像素占比 < 1% (近乎全白/全空) → `blank_render` (相机/尺度问题或空模型)
- 30 视角图像间方差极低 (几乎全同) → `low_variance` (高度对称简单体, pretrain 价值低)

### 方面 4: 语义/去重级筛选 (依赖 CLIP embedding, 后期)

- 用预训练 CLIP embedding 做近重复去重 (余弦相似度阈值)
- CLIP 空间分布做多样性采样 (避免 pretrain 子集过度集中某类零件)

### 待拍板的方向性决策

检索任务面向 "零件级" 还是 "装配体级"? 这定义了 complexity 是要筛掉的噪声还是要特殊处理的信号:
- 仅单零件: n_solids > 1 装配体可直接筛掉/拆解, 复杂度问题大半消失
- 保留装配体: 高 n_faces 需单独慢速渲染队列 (更长 timeout + 降网格精度), 而非筛掉

### 落地

写独立 probe.py: 读 manifest pending 行, 做方面 1+2 探查, 写回 probe_* 列和 quality_flags,
完全不 mesh/不渲染。跑完首次得到 ABC 几何复杂度真实分布, 阈值看直方图定 (而非拍脑袋的 340K)。
manifest 新增列: probe_n_faces, probe_n_solids, probe_bbox_ratio, file_size_mb, quality_flags
渲染阶段只取 render_eligible (无致命 flag) 的行。
