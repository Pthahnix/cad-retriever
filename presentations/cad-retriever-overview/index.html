核心设计决策

决策点选择为什么
CAD 侧融合α·BRep + (1-α)·MV离线计算，简单可靠，α=0.7 偏向拓扑
Query 侧融合Cross-Attention文本约束图示语义（"这个形状，但要三个孔"）
单模态处理跳过融合层纯文本 ~0.8ms，不引入不必要的计算
Embedding 维度512 维统一空间平衡表达力与检索效率，FAISS 索引 ~2GB
检索粒度零件级工程实际需求，装配体太粗、特征级太细
Query 类型文本 / 图示 / 混合覆盖所有实际使用场景

Encoder 设计 — CAD 侧

B-Rep Transformer（主路径）

架构6 层 Transformer，dim 512，8 heads
输入B-Rep 序列化 token（face-edge 交替）
位置编码拓扑感知 PE（邻接矩阵相对偏置）
序列上限256 tokens
输出[CLS] → projection → 512 维

为什么选 Transformer 而非 GNN？

  • 与 CLIP 文本塔架构对称，训练对齐自然
  • 固定长度序列易 batch 化，GPU 利用率高
  • 全局注意力捕获长距离拓扑依赖
  • 复用 TensorRT Transformer 优化栈

多视图 ViT（辅助路径）

架构CLIP ViT-B/16，冻结主体
输入6 标准视角渲染图 224×224
训练只训练最后 2 层 + projection
聚合6 图 mean pooling → 512 维
作用补充视觉外观语义

融合公式：

CAD_vec = 0.7 × brep_emb + 0.3 × mv_emb

α 可学习，初始化偏向 B-Rep 拓扑信息

5

Encoder 设计 — Query 侧

Text Encoder

  • Long-CLIP 文本塔（248 token）
  • 12 层 Causal Transformer
  • 冻结 + LoRA rank=16
  • [EoT] → projection → 512 维
  • 推理版: TinyBERT-4L 蒸馏

解决 77 token 限制，CAD 描述需要长文本

Image Encoder

  • CLIP ViT-B/16
  • 全量 fine-tune
  • 输入: 224×224 单张图
  • [CLS] → projection → 512 维
  • 统一处理三种图示类型

草图 / 工程图 / 渲染截图一个模型通吃

Cross-Attention 融合

  • 2 层 Cross-Attn + 1 层 Self-Attn
  • Text tokens 为 Q
  • Image tokens 为 K, V
  • 融合 [CLS] → 512 维 query
  • 单模态时完全跳过

文本约束图示："这个形状，但要圆孔"

训练策略 — 四阶段流程

Phase 1

基础对齐 (3-5天)

  • CAD ↔ Text InfoNCE 双向对比
  • Batch 2048 (gradient cache)
  • τ 可学习，初始 0.07
  • Text 冻结+LoRA / B-Rep 全量

Phase 2

图示对齐 (1-2天)

  • Image → CAD 对比损失
  • 复用 Phase 1 CAD embedding
  • 渲染图 + 工程图 + 合成草图
  • Image Encoder 全量 fine-tune

Phase 3

融合训练 (1天)

  • Cross-Attn 融合层训练
  • 三元组 (text, sketch, CAD)
  • 联合 loss 防单模态退化
  • 验证: 融合 > 单模态

Phase 4

知识蒸馏 (0.5天)

  • Teacher: 12 层 → Student: 4 层
  • 中间层对齐 + embedding MSE
  • 门槛: Recall@10 下降 ≤2%
  • 否则回退 6 层

Hard Negative Mining（精度提升关键手段）

每 epoch 用当前模型检索 Top-50，取 rank 10-30 作为下轮 hard negative。挖掘两类：几何相似但语义不同（M6 vs M8 螺栓）、语义相似但几何不同（焊接法兰 vs 螺栓法兰）

FP8 Tensor Core

Blackwell 原生 FP8 推理

  • 比 INT8 精度更高（指数位保留）
  • 速度与 INT8 相当
  • TensorRT 9.x 原生支持
  • 优先于 INT8 使用
🧩

Sparsity 2:4

结构化稀疏加速

  • 每 4 个权重保留 2 个非零
  • 额外 ~1.5× 加速
  • 需要稀疏感知训练
  • 与 FP8 可叠加使用
🔍

CAGRA 图索引

FAISS 1.10 + cuVS

  • GPU 原生图索引算法
  • 比 IVFFlat 快 3×
  • 1M 规模 ~0.1ms
  • 召回率 ~98%（vs IVFFlat 95%）

32GB VRAM 余量充足：模型 + 索引仅占 ~3.3GB，剩余 28.7GB 可用于更大索引、多模型并行、或未来扩展到 10M 规模

风险与 Fallback

每个风险都有明确的退化路径，系统不会完全失效

B-Rep Transformer 训练不稳定

主路径失效

→ 多视图路径兜底，系统仍可用

Cross-Attention 延迟超标

双模态超 10ms

→ 退化为 late fusion（加权平均）

ViT-B/16 延迟超标

图示检索超标

→ 换 ViT-S/16 (22M) 或降分辨率

TinyBERT 精度损失过大

Recall 下降 >3%

→ 回退到 6 层蒸馏

5090 TensorRT FP8 不成熟

编译失败

→ 回退 FP16 + INT8 组合

草图合成数据分布偏差

草图检索差

→ 收集少量真实草图 fine-tune
